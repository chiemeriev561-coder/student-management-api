# Contributing

## Development Setup

```bash
git clone https://github.com/chiemeriev561-coder/student-management-api.git
cd student-management-api
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
alembic upgrade head
```

Create a `.env` file with `DATABASE_URL` and `AUTH_SECRET_KEY` before running the app locally.

## Workflow

1. Branch from `main`.
2. Make one focused change at a time.
3. Add or update tests with the change.
4. Run the test suite before opening a pull request.
5. Add an Alembic migration for schema changes.

## Coding Standards

- Follow the existing FastAPI, SQLAlchemy, and Pydantic patterns already used in `app/`.
- Keep functions small and behavior-oriented.
- Prefer explicit names over short abbreviations.
- Avoid unrelated refactors in the same pull request.
- Preserve backward compatibility unless the change is intentionally breaking and documented.

## Testing Requirements

Run:

```bash
python -m pytest -q
```

Useful subsets:

```bash
python -m pytest tests/unit -q
python -m pytest tests/integration -q
```

Expectations:

- New features should include tests.
- Bug fixes should include a regression test when practical.
- Database changes should be covered by migration checks and API or CRUD tests.

## Database Changes

When you modify SQLAlchemy models:

```bash
alembic revision --autogenerate -m "describe the change"
alembic upgrade head
```

Review autogenerated migrations before committing them.

## Pull Requests

Include:

- What changed
- Why it changed
- Any migration steps
- Any breaking API behavior
- Test evidence, usually the `pytest` command result
